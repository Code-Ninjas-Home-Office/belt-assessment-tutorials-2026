# Orange Belt Assessment

## Orange Belt Assessment @showdialog 
To show you are ready for Green Belt, complete the final Prove Yourself of Orange Belt to demonstrate your understanding of the concepts taught in Orange Belt, including sprite kinds, variables, arrays and nested arrays, functions, conditionals in nested block statements, for loops, and the Story extension.

Create a project that fulfills all of the requirements on the left side. Click Next and alert a Code Sensei when you are ready to have your project checked.

Click **Ok** to get started!

## Requirements

A complete Belt Assessment must meet the following requirements: 

- :paint brush: Create 2 **tilemaps** with starting and ending tiles for an Enemy sprite, and obstacles for the Player sprite to move around. 
- :function: Use a **function** with a **parameter** to create at least 2 levels in the game. Call the **function** to go to the next level’s tilemap when the score equals a specific number. Display the level number with a dialog box that announces which level number it is. End the game once the user has scored a certain number of points.
- :paper plane: Create a **Player sprite** that moves with the direction buttons and is followed by the camera. Set the Player sprite’s position to a random location in the first row or column of the tilemap in the function that sets the level tilemaps. Reset the Player sprite’s position after overlapping with an Enemy sprite.
- :function: Use a **function** to create a Food sprite that appears on a random tile for a random length of time. Call the **function** to make a new Food sprite when the existing one is destroyed.
- :function: Use a **function** to create an Enemy sprite that starts on the opposite side of the screen, moves across the screen, and can “ghost” through the tilemap walls. Destroy the Enemy sprite when it overlaps the Player sprite or the ending tile. Call the **function** to make a new Enemy sprite when the existing one is destroyed.
